Mes courses — refonte (3 onglets + Hero Earnings + cards améliorées)

Maquettes statiques pour valider la direction avant le code. Audit complet de la page actuelle, propositions priorisées : Hero Earnings permanent en haut, fusion Agenda/À venir, cards d’urgence pulsantes, filtres avancés Historique, facture PDF par trimestre, empty state éducatif.

Décisions de design retenues

  • A Hero Earnings permanent au-dessus des onglets — gain du jour, sparkline 7j, objectif quotidien. Tap → historique filtré aujourd’hui.
  • B 3 onglets au lieu de 4 : « À venir », « Postées », « Historique ». L’Agenda devient un toggle d’affichage dans À venir (Liste / Semaine / Mois).
  • C Card prochaine course enrichie : nom patient + heure + trajet + 2 boutons (Naviguer / Appeler) en un coup d’œil.
  • D Pulse rouge sur courses < 15 min, position épinglée en haut.
  • E Postée > 24h sans preneur : strip d’action (élargir aux groupes voisins / +5 € sur le prix).
  • F Historique enrichi : 4 KPI tiles, filtres type+groupe+période, summary trimestre avec bouton « Facture PDF ».
  • G Empty state éducatif sur premier login : 3 cards explicatives.

Détails techniques à valider en parallèle

  • Hero Earnings : nouveau composant CoursesEarningsHero.tsx alimenté par un nouveau service earningsService.getDailyStats(). Calculé depuis missions WHERE status='COMPLETED' AND completed_at >= today. Sparkline = 7 jours derniers, agrégés côté serveur (vue Postgres ou requête).
  • Mode Liste/Semaine/Mois sur À venir : on garde la logique useAgendaTab mais on supprime l’onglet dédié. Le toggle de vue est dans useUpcomingTab. Migration douce (3 onglets actuels → 4 onglets pendant 2 semaines avec banner d’info, puis 3 onglets).
  • Pulse rouge urgent : déjà animation pulse-urgent dans le mockup. À porter dans UpcomingMissionCard avec condition minutesUntil <= 15.
  • Strip boost : nouveau widget visible sur PostedCard si posted_at < now() - 24h && status === 'AVAILABLE'. Actions : élargir = ajouter d’autres groupes, +5 € = update price_eur.
  • Filtres type/groupe historique : useHistoryTab accepte 2 nouveaux filtres optionnels (type, groupId). Reste compatible avec l’existant.
  • Facture PDF : nouvelle API route /api/missions/invoice qui génère un PDF avec pdfkit ou @react-pdf/renderer. Inclut TVA, n° SIREN du chauffeur, tampon TaxiLink. Quart calculé depuis le mois courant.
  • FAB mobile : déplacer le FAB de l’Agenda vers le niveau page (toujours visible). Cohérent avec l’Accueil qui n’en a pas.
  • Empty state éducatif : composant CoursesEmptyOnboarding.tsx affiché si missions.length === 0 && completed.length === 0 && posted.length === 0. Disparaît dès la 1re action.

Si tu valides ces directions, plan d’exécution

  • Sprint 1 (3-4h) — Hero Earnings + nouveau service stats + sparkline. C’est le truc à plus gros impact perçu.
  • Sprint 2 (2-3h) — Pulse rouge urgent + Hero prochaine course enrichi (patient + heure + appel) + FAB mobile global.
  • Sprint 3 (3-4h) — Fusion onglet Agenda dans À venir avec toggle Liste/Semaine/Mois. Migration douce.
  • Sprint 4 (2-3h) — Filtres type/groupe historique + KPI tiles + summary trimestre.
  • Sprint 5 (4-5h) — Facture PDF (nouvelle API route + pdf lib).
  • Sprint 6 (1-2h) — Empty state éducatif + strip boost Postées > 24h.
