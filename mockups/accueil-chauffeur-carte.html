Accueil chauffeur — annonces géolocalisées (3 pistes)

Maquettes statiques à valider avant tout code. Style Inter + tokens dashboard existants (jaune #FFD23F, noir #1A1A1A, arrondis 16/20/24). Pins = annonces, pas voitures.

Éléments communs aux 3 variantes

  • Carte = annonces autour du chauffeur. Pin = prix course. Pin sélectionné = inversé (noir/jaune).
  • Position chauffeur = point bleu avec halo (au centre par défaut, repositionné en live via Geolocation API).
  • Liste = adresse départ, destination, type (CPAM/Privé/Prescription), prix, distance depuis la position chauffeur.
  • Tap pin ↔ tap carte liste : les deux se synchronisent (même state selectedMissionId).
  • Bouton CTA : "Voir les annonces" si rien sélectionné → "Maintenir 2s pour accepter" (barre de progression jaune) + bouton rond "Détail" (icône info) quand une annonce est active.
  • Filtres : chips horizontales défilables — Type (CPAM/Privé/Prescription), Urgence (< 10 min), Distance (< 5 / 10 / 20 km), Aller-retour.

Détails techniques à valider en même temps

  • Carte : Mapbox GL JS (déjà NEXT_PUBLIC_MAPBOX_TOKEN prévu) ou Leaflet + OSM (gratuit, déjà utilisé pour OSRM). Ma reco : Mapbox si la clé est dispo, sinon Leaflet.
  • Géoloc chauffeur : navigator.geolocation.watchPosition, stocké dans un store Zustand driverLocationStore. Poussé en Supabase realtime toutes les 10-30 s (à discuter pour ne pas exploser la quota).
  • Géoloc annonces : ajouter departure_lat / departure_lng sur missions (migration). Déjà résolus lors du PartagerMissionModal via BAN. Backfill des missions existantes via addressService.
  • Filtrage par rayon : PostGIS ou simple formule Haversine côté client sur la liste déjà filtrée par département.
  • Hold 2s pour accepter : onPointerDown démarre un timer 2 s + animation CSS remplissage, onPointerUp avant 2 s → cancel. À 2 s → missionStore.acceptMission(). Anti-tap accidentel.
  • Bouton Détail : ouvre MissionDetailScreen (déjà existant) en modal bottom sheet.
  • État vide : si 0 annonce → message type "Aucune annonce autour de vous" + bouton "Élargir le rayon" et rappel des dept_preferences.
  • Annonce acceptée : retirer la card « Félicitations » de MissionAcceptedCelebration.tsx. Garder uniquement le pouce vert animé centré + ConfettiOverlay, z-index ≥ 60 pour passer au-dessus de la carte. Confettis non bloqués → arrivent immédiatement.
  • Annonce publiée : remplacer/compléter l'écran MissionPublishedStep.tsx par un toast discret affiché sur DriverHome au retour du modal. Fond --secondary, texte --primary. Contenu : type (CPAM/Privé/Prescription), destination courte, prix. Auto-dismiss 3-4 s. Probablement à brancher dans useToasts() avec un variant "publish".
